# SectorFlow AI Work Rules

> [!CAUTION]
> **모든 AI(메인 및 하위 에이전트 포함)는 아래의 작업 수칙을 최우선으로 준수해야 합니다.**

## 1. 사전 승인 없는 코드 수정 절대 금지 (Strict Planning Mode)
- 아무리 사소한 버그 수정, 예외 처리 추가, 오타 수정이라 할지라도 **사용자(USER)의 명시적인 승인(Approval)이 떨어지기 전까지는 절대로 소스 코드를 임의로 수정(Write/Modify)해서는 안 됩니다.**
- 오직 코드 조사(Research)와 읽기(Read)만 수행한 뒤, 반드시 `implementation_plan.md`를 작성하여 사용자에게 제출하고 대기해야 합니다.

## 2. 아키텍처 및 의존성 존중
- 데이터베이스(DB)는 보조 캐시(Soft Dependency) 역할만을 수행하며, DB 장애나 데이터 부재가 애플리케이션의 크래시(Crash)나 기동 실패로 이어져서는 안 됩니다.
- 모든 스키마 및 아키텍처 변경은 철저히 전체 파이프라인(저장-로드-UI표시)을 검증한 뒤 계획서에 담아야 합니다.

## 3. 실행 단계 (Execution)
- 사용자의 "승인(시작해, 진행해 등)"이 떨어진 직후에만 코드를 수정합니다.
- 수정이 완료되면 즉시 변경 사항을 사용자에게 보고하고 테스트를 요청합니다.
