# [Phase 1] 다중 증권사 전환 어댑터 설계 및 UI 범용화 계획서

## 1. 개요 및 목표
현재 프론트엔드 UI에 '키움증권'으로 하드코딩된 종속성을 제거합니다. 사용자가 키움증권과 LS증권을 자유롭게 선택하여 시스템 전체의 통신망을 전환할 수 있는 구조를 만들고, UI가 이에 맞춰 범용적으로 동작하도록(Adapter 패턴 도입) 개선합니다.

## 2. 심층 사전조사 결과 (영향성 평가)
- **백엔드 통신망 라우터 (`broker_router.py` 등) 조사 완료:**
  백엔드는 이미 다중 증권사 라우팅이 완벽하게 구현되어 있습니다. `settings.json` 내의 **`broker`** (기본값 "kiwoom") 필드 하나만 변경하면, 시스템 전체의 주문(Order), 시세(Quote), 업종(Sector) 데이터 소스가 자동으로 해당 증권사로 일괄 전환됩니다. 또한 모든 증권사의 API 키(`_app_key`, `_app_secret` 등)를 알아서 동적으로 보관하도록 준비되어 있습니다.
- **헤더 상태 표시 (`header.ts`):** 
  현재 하드코딩된 `kiwoom_token_valid`, `kiwoom_connected` 필드에 의존하고 있습니다. 이를 **현재 활성화된 주 사용 증권사(`settings.broker`)** 기준으로 뱃지 이름과 색상이 동적으로 변경되도록 개선해야 합니다.

## 3. 구현 계획 및 명확한 동작 정의

사용자님의 지적을 반영하여 설계의 불분명한 점을 완전히 해소했습니다.

### ① '주 사용 증권사(Primary Broker)' 선택과 'API 키 입력'의 완전한 분리
일반설정(`general-settings.ts`) 화면을 두 가지 역할로 명확히 나눕니다.
- **역할 A (통신망 전환):** 최상단에 **[주 사용 증권사 선택]** 라디오 버튼(키움증권 / LS증권)을 배치합니다. 이 값을 변경하면 백엔드의 `broker` 설정값이 바뀌며, **실시간 시세, 계좌 조회, 주문망 등 시스템의 통신 핏줄이 즉시 해당 증권사로 일괄 전환**됩니다. 
  *(단, 업종 분류 데이터는 사용자 커스텀으로 관리되므로, 증권사를 변경하더라도 사용자가 직접 커스텀한 업종 분류 체계는 그대로 독립적으로 유지됩니다.)*
- **역할 B (API 키 관리):** 그 아래에 있는 'API 설정' 탭은 단순히 각 증권사의 **키를 입력하고 저장해두는 보관함** 역할만 합니다. 즉, LS증권 탭을 눌러서 키를 입력한다고 해서 시스템이 LS증권으로 전환되는 것이 아닙니다. 

### ② 헤더 UI 뱃지의 상태 표시 기준
헤더 좌측 상단에 뜨는 상태 뱃지는 **[주 사용 증권사] 로 선택된 단 하나의 증권사 상태**만 대표로 표시합니다.
- 예: 주 증권사가 LS증권일 경우, 백엔드가 LS증권과 연결되면 헤더 뱃지가 빨간색의 "LS증권", "LS실시간"으로 켜집니다. 키움증권이 뒤에서 몰래 연결되어 있더라도 헤더에는 메인 통신망(LS)의 상태만 직관적으로 표시됩니다.

## 4. 검증 계획
- **통신망 전환 테스트:** 설정에서 주 사용 증권사를 LS로 변경했을 때, 헤더 뱃지가 즉시 LS로 변경되고 백엔드 로그에 라우터 전환(`broker=ls`)이 찍히는지 확인.
- **API 키 독립 저장 테스트:** 키움증권 탭과 LS증권 탭 양쪽 모두에 API 키를 다르게 입력해도 서로 간섭 없이 온전히 보관되는지 확인.
