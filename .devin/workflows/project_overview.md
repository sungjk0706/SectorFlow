## 프로젝트 목표
1인 로컬 실시간 데이터 처리 기반 주식 자동매매 웹앱. 사용자가 설정한 조건에 따라 업종 순위를 계산하고, 상위 업종의 종목만 자동 매수/매도.

## 매매 파이프라인 (장마감 → 다음 거래일)

### 1. 데이터 수집 (장마감 후)
- 매매적격종목들의 5일봉 거래대금, 고가 다운로드
- 다음 거래일 매매 로직 지표로 사용

### 2. 1차 필터링 (종목 단위)
- 5일봉 평균거래대금 N억원 이상 종목만 통과 (사용자 설정값)

### 3. 업종 그룹화
- 통과한 종목들을 사용자가 커스텀한 업종으로 그룹화

### 4. 업종 스크리닝
- 업종 내 상승종목 비율이 사용자 설정 N% 이하 → 업종 순위 탈락
- 업종 내 상승종목 비율이 사용자 설정 N% 이상 → 통과

### 5. 업종 순위 계산
- 통과한 업종의 거래대금을 N종목으로 나누어 거래대금 평균 계산
- 거래대금 평균 + 상승종목 비율 → 가중치 계산으로 업종 순위 산출
- 사용자가 설정한 상위 N개 업종만 통과

### 6. 매수 후보 등록
- 통과한 업종에 속한 종목들 → 매수설정 페이지 매수후보 테이블에 진입

### 7. 매수 실행
- 매수후보 종목이 사용자가 설정한 매수설정 값에 부합할 때만 매수주문
- 주문 방식: 시장가

### 8. 매도 관리
- 체결되면 매도설정 페이지 보유종목 테이블로 이동
- 사용자가 설정한 매도설정 값에 부합할 때 매도 실행

## 사용자 설정값 (모든 N값은 사용자 커스텀)
- 5일봉 평균거래대금 최소 기준 (억원)
- 업종 내 상승종목 비율 하한/상한 (%)
- 업종 순위 가중치 방식
- 상위 업종 수 (N개)
- 매수설정 조건
- 매도설정 조건

## 데이터 흐름 요약
```
장마감 → 5일봉 다운로드 → 종목 필터링(거래대금) → 업종 그룹화 → 업종 스크리닝(상승비율) → 업종 순위(가중치) → 상위 N업종 → 매수후보 테이블 → 매수실행(시장가) → 보유종목 테이블 → 매도실행
```

## 주요 프론트엔드 파일
- `stores/store.ts`: Zustand 전역 상태 (Record 기반)

## 기술 스택
- 백엔드: FastAPI (Python)
- 프론트엔드: TypeScript (Vanilla)
- 데이터베이스: SQLite
