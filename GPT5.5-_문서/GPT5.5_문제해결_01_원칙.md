# AI 문제해결 참고서 01 - 근본해결 원칙

## 1. 근본해결이란 무엇인가

근본해결은 증상을 잠시 숨기는 것이 아니라, 같은 문제가 다시 발생하지 않도록 원인 경로를 제거하는 것이다.

예를 들어 UI가 보이지 않을 때 다음은 근본해결이 아니다.

```text
z-index를 무작정 올림
!important 추가
height를 여기저기 임의로 추가
새로고침만 안내
임시로 display 값을 바꿈
```

근본해결은 다음에 가깝다.

```text
왜 보이지 않는지 코드상 렌더링 여부 확인
props/state 전달 경로 확인
CSS 충돌 경로 확인
flex 스크롤 체인 확인
캐시/서비스워커 여부 확인
원인별로 충돌 경로 제거
```

---

## 2. 코드 기반 조사 원칙

하위 AI는 추측으로 결론 내리면 안 된다.

반드시 다음을 근거로 삼는다.

- 실제 파일 내용
- 검색 결과
- 빌드 로그
- 런타임 로그
- 브라우저에서 재현 가능한 현상
- store/state/data flow 확인 결과

금지 표현:

```text
아마도
그럴 것 같습니다
일단 해보죠
```

허용 표현:

```text
파일 A의 n번째 줄에서 확인됨
검색 결과 해당 클래스가 남아 있음
빌드 로그에서 타입 오류가 확인됨
이 부분은 아직 확인 필요
```

---

## 3. 보고와 수정의 분리

사용자가 “원인 분석 후 보고하라”고 하면 수정하면 안 된다.

올바른 순서:

1. 코드 조사
2. 원인 보고
3. 해결 방안 제시
4. 사용자 승인 확인
5. 수정
6. 검증
7. 결과 보고

사용자가 명시적으로 “진행해”, “수정해”, “근본해결책으로 진행해”라고 승인한 경우에만 수정한다.

---

## 4. 작은 단위 진행 원칙

큰 명령이나 대규모 자동 치환은 사용자가 진행 여부를 알기 어렵다.

따라서 다음 원칙을 지킨다.

- 파일 하나씩 수정한다.
- 큰 파일은 블록 단위로 수정한다.
- 수정 후 바로 검색 검증한다.
- 중간 상태를 짧게 보고한다.
- 오래 걸릴 수 있는 명령은 사전에 알린다.

나쁜 방식:

```text
긴 python 치환 스크립트로 여러 파일 한 번에 변경
큰 apply_patch로 반복 문자열을 무리하게 치환
정규식 하나에 많은 조건을 섞어 검색
```

좋은 방식:

```text
파일 읽기
작은 패치 적용
고정 문자열 검색으로 잔여 확인
다음 파일 진행
빌드 검증
```

---

## 5. 실패를 줄이는 검색 원칙

복잡한 정규식은 오류를 만들 수 있다.

특히 다음 문자는 정규식에서 주의한다.

```text
{ } ( ) [ ] . * + ? | \
```

안전한 검색 방식:

- 정확한 문자열은 고정 문자열 검색을 사용한다.
- 여러 조건을 한 번에 검색하지 말고 나눠서 검색한다.
- 검색 결과가 많으면 범위를 좁힌다.

예:

```text
className="setting
className="input-small"
.main-container
.setting-row
```

---

## 6. 검증 없는 완료 보고 금지

수정 후 다음 중 최소 하나 이상을 확인해야 한다.

- 잔여 문자열 검색
- 타입 체크
- 빌드
- 테스트
- 브라우저 재현 확인

프론트엔드 변경의 기본 검증:

```bash
npm run build
```

단, 빌드 성공만으로 브라우저 표시가 100% 보장되는 것은 아니다. UI 문제는 실제 화면 확인이 필요하다.

---

## 7. 임시방편 금지 목록

다음 방식은 특별한 이유 없이 사용하지 않는다.

- `!important` 남발
- 타입 오류를 무조건 `as any`로 덮기
- 전역 CSS에 페이지 전용 스타일 추가
- service worker 캐시 문제를 무시하고 코드만 계속 수정
- 사용자 승인 없이 범위 밖 리팩토링
- UI 디자인을 임의 변경

---

## 8. 완료 보고 형식

완료 보고에는 다음을 포함한다.

```text
수정한 파일
해결한 근본 원인
검증한 항목
남은 확인 사항
사용자가 직접 확인할 방법
```

예:

```text
매수설정/매도설정 CSS 클래스명 충돌 제거 완료
flex scroll chain 수정 완료
잔여 전역 클래스 검색 결과 없음
npm run build 성공
브라우저에서 좌측 설정 스크롤 확인 필요
```
