# 백엔드 로그 한글화 2차 작업 계획서

> **작성일**: 2026-07-12
> **목적**: 백엔드 로그 중 잔존 영어/기술용어/영어 키값을 한국 주식시장 일반 사용자가 이해할 수 있는 표현으로 수정
> **범위**: 사용자 직접 노출 로그 우선 (약 30개 파일, 200건+). 내부 디버그 로그는 2차 작업으로 미룸.

---

## 수정 원칙

### 1. 사용자 노출 우선
- 웹서버 기동/종료, 매매, 계좌, 오류 알림 로그를 1차 작업 대상으로 함
- 내부 디버그 로그(pipelines, db 마이그레이션, engine_cache 등)는 2차 작업으로 미룸

### 2. API 코드 → 기능명+코드 병기
- 예: `ka10099` → `전종목 통합 조회(ka10099)`
- 사용자 이해도 향상 + 개발자 추적 용이성 동시 확보

### 3. 고유명사 유지
- **한국 주식시장 고유명사**: KRX, NXT
- **증권사명**: 키움, LS
- **기술 표준명**: HTTP, JSON, REST, DB, API, PK, PID, SIGTERM, OAuth
- **프로토콜/라이브러리명**: WebSocket(단, 사용자 노출 로그에서는 "실시간 통신"으로 치환)

### 4. 내부 디버그 로그 2차 미룸
- `pipelines/pipeline_compute.py`, `pipelines/pipeline_gateway.py`의 내부 틱 처리 로그
- `db/stock_tables.py`의 마이그레이션 로그
- `engine_cache.py`, `engine_snapshot.py`의 내부 캐시 로드 로그
- `core/sector_mapping.py`, `core/sector_stock_cache.py`의 내부 조회 로그
- `core/trading_calendar.py`, `core/lock_manager.py`, `core/journal.py`, `core/memory_monitor.py`
- `core/settings_file.py`, `core/settings_store.py`
- `services/sector_data_provider.py`, `services/ws_subscribe_control.py`, `services/engine_strategy_core.py`
- `services/engine_ws_fill_followup.py`, `services/core_queues.py`
- `services/market_close_pipeline.py` (대부분 내부 파이프라인)
- `services/engine_ws_reg.py`, `services/engine_ws.py` (구독 내부 로직)
- `services/engine_sector_confirm.py`, `services/engine_config.py`
- `services/data_manager.py` (debug 레벨)
- `web/routes/` 중 logger 호출 없는 파일들

### 5. 맞춤법/띄어쓰기 통일
- `예외` → `오류` (사용자 노출 로그 한정)
- `%d회` → `%d번`
- `스킵` → `생략`/`건너뜀`
- `브로드캐스트` → `전송` (사용자 노출 로그 한정)
- `폴백` → `대체`

- `앱준비` → `앱 준비`

- `설정채널` → `설정 채널`
- `체결채널` → `체결 채널`
- `이미구독` → `이미 구독`
- `계좌등록` → `계좌 등록`
- `계좌해제` → `계좌 해제`
