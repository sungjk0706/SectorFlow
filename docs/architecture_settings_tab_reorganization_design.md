# 설계서: 일반설정 페이지 탭 재구성 (Settings Tab Reorganization)

> **상태**: 설계 완료 · 구현 승인 대기
> **작성일**: 2026-07-18
> **다단계 워크플로우**: 1세션(본 설계서) → 2세션(태스크 파일) → 3세션~(단계별 구현)
> **관련 원칙**: P10(SSOT) · P16(살아있는 경로) · P20(폴백 금지) · P21(사용자 투명성) · P23(일관성) · P24(단순성)
> **대상 파일**: `frontend/src/pages/general-settings.ts` (주), 공통 컴포넌트 재사용 검토

---

## 1. 배경 및 목적

### 1-1. 문제 상황 (검토 결과 요약)

일반설정 페이지는 4개 탭(자동매매 / 투자모드 / 텔레그램 / API 설정)으로 구성되나, 최근 추가된 기능(체결 불가 시간대 주문 차단, 장 시작 전 사전 준비 시간 3개 슬롯, 거래소 고정 시간 참고 표시)이 자동매매 탭과 API 설정 탭에 혼재되어 아래 문제가 발생.

| 문제 | 상세 | 위반 원칙 |
|---|---|---|
| **시간 설정 분산** | "시간"이라는 동일 주제가 자동매매 탭(자동매수/매도 시간쌍, 타임테이블 3개 슬롯, 거래소 고정 시간 참고)과 API 설정 탭(1일봉차트 자동다운로드 시간 20:40) 양쪽에 흩어짐. 사용자가 시간을 바꾸려면 두 탭을 뒤져야 함 | P23(일관성) |
| **자동매매 탭 과밀** | 자동매매 탭 하나에 6개 섹션·10개 이상 설정 행이 몰려 있음. "장 시작 전 사전 준비 시간(3개 슬롯) + 거래소 고정 시간 참고(7개 행)"은 자동매매 ON/OFF와 직접 관련 없는 시스템 시간 설정인데 자동매매 탭에 섞여 있어 시각적 부담 | P24(단순성) |
| **API 설정 탭 이질 요소 혼재** | "1일봉차트 자동다운로드"(시간/스케줄러 설정)와 "실시간 현재가 플래시 효과"(UI 표시 설정)가 API 키 보관 탭에 위치. 탭의 본래 목적(증권사 선택 + API 키 보관)과 불일치 | P23(일관성) |
| **거래소 고정 시간 발견성 낮음** | 거래소 고정 시간 참고(08:00~20:00, 7개)가 자동매매 탭 하단에 있어 사용자가 "내가 설정한 시간"과 "거래소 고정 시간"을 비교하려면 스크롤해야 함 | P21(사용자 투명성) |

### 1-2. 목적

- **시간 설정 단일 소스화**: 모든 "시간" 설정을 한 탭에 모아 사용자가 한 화면에서 시간 흐름(07:58 → 08:59 → 09:00 → 15:20 → 20:40)을 한눈에 파악
- **자동매매 탭 단순화**: 매매 ON/OFF 스위치 중심으로 축소, 시스템 시간 설정 분리
- **API 설정 탭 본래 목적 회복**: 증권사 선택 + API 키 보관만 남김
- **거래소 고정 시간 발견성 강화**: 사용자 설정 시간과 거래소 고정 시간을 나란히 배치 (P21)
- **탭 주제 일관성 확보**: 탭당 단일 주제, 행 수 축소 (P23/P24)

## 2. 사용자 확정 사항

> 본 설계서는 아래 4항목 사용자 확정을 전제로 작성됨.

### 2-1. "시간 설정" 탭 신설

신규 탭 `time-settings` 추가. 다음 구성요소를 한 탭에 통합:

| 구성요소 | 이동 출처 | 백엔드 키 |
|---|---|---|
| 자동매수 시간쌍 (시작/종료) | 자동매매 탭 | `buy_time_start`, `buy_time_end` |
| 자동매도 시간쌍 (시작/종료) | 자동매매 탭 | `sell_time_start`, `sell_time_end` |
| 실시간 항목 초기화 시간 | 자동매매 탭 | `timetable.realtime_reset` |
| 구독 사전 시작 시간 | 자동매매 탭 | `timetable.ws_prestart` |
| 정규장 사전 구독 시간 | 자동매매 탭 | `timetable.krx_pre_subscribe` |
| 1일봉차트 자동다운로드 시간 + 토글 | API 설정 탭 | `confirmed_download_time`, `scheduler_market_close_on` |
| 거래소 고정 시간 참고 표시 (읽기전용) | 자동매매 탭 | (코드 상수 — 변경 불가) |

### 2-2. 자동매매 탭 정리

자동매매 탭에는 매매 ON/OFF 스위치만 유지:

| 구성요소 | 백엔드 키 | 비고 |
|---|---|---|
| 자동매매 마스터 토글 | `time_scheduler_on` | 비거래일 뱃지 표시 유지 |
| 자동매수 ON/OFF 토글 | `auto_buy_on` | 시간쌍 입력은 시간 설정 탭으로 이동 |
| 자동매도 ON/OFF 토글 | `auto_sell_on` | 시간쌍 입력은 시간 설정 탭으로 이동 |
| 체결 불가 시간대 주문 차단 토글 | `order_time_guard_on` | 현행 유지 |

### 2-3. API 설정 탭 정리

| 구성요소 | 처리 |
|---|---|
| 주 사용 증권사 라디오 (키움/LS) | 유지 |
| 키움 API / LS API 서브탭 (앱키·앱시크릿·계좌번호) | 유지 |
| 1일봉차트 자동다운로드 (토글 + 시간 슬롯) | **시간 설정 탭으로 이동** |
| 실시간 현재가 플래시 효과 토글 | **자동매매 탭 하단으로 이동** (새 "화면 설정" 탭 신설은 사용자가 과잉으로 판단하여 기각) |

### 2-4. 탭 순서 확정

```
자동매매 → 시간 설정 → 투자모드 → API 설정 → 텔레그램
```

- 자동매매를 최상단에 두어 "전체 스위치"를 먼저 확인 (사용자 최우선 관심사)
- 시간 설정을 두 번째에 두어 자동매매 ON 직후 시간 구성을 자연스럽게 이어서 확인
- 투자모드(API 키와 무관하게 테스트/실전 스위치)를 API 설정보다 먼저 배치
- 텔레그램은 부가 기능이므로 최하단

## 3. 재구성 전후 비교

### 3-1. 탭 구성 비교표

| 탭 | 재구성 전 | 재구성 후 |
|---|---|---|
| **자동매매** | 마스터 토글, 자동매수(토글+시간쌍), 자동매도(토글+시간쌍), 체결 불가 시간대 차단, 장 시작 전 사전 준비 시간(3개 슬롯), 거래소 고정 시간 참고 | 마스터 토글, 자동매수 토글, 자동매도 토글, 체결 불가 시간대 차단, **실시간 현재가 플래시 효과** (API 설정 탭에서 이동) |
| **시간 설정** | (존재하지 않음) | **신설**: 자동매수 시간쌍, 자동매도 시간쌍, 장 시작 전 사전 준비 시간(3개 슬롯), 1일봉차트 자동다운로드(토글+시간), 거래소 고정 시간 참고 |
| **투자모드** | 투자모드 라디오, 가상 투자금 섹션 | (변경 없음) |
| **API 설정** | 주 사용 증권사, 1일봉차트 자동다운로드, 실시간 현재가 플래시 효과, 키움/LS API 키 서브탭 | 주 사용 증권사, 키움/LS API 키 서브탭 |
| **텔레그램** | 알림 토글, 채팅 ID/봇 토큰, 명령어 안내 | (변경 없음) |

### 3-2. 시간 설정 탭 내부 순서 (사용자 관점 시간 흐름)

사용자가 하루 시간 흐름을 직관적으로 파악할 수 있도록 시간순 배치:

```
[시간 설정 탭]
├─ 자동매수 시간 (시작/종료)        ← 09:00~15:00 (사용자 조정)
├─ 자동매도 시간 (시작/종료)        ← 09:00~15:00 (사용자 조정)
├─ ─────────────────────────────
├─ 장 시작 전 사전 준비 시간
│   ├─ 실시간 항목 초기화           ← 07:58 (timetable.realtime_reset)
│   ├─ 구독 사전 시작               ← 07:59 (timetable.ws_prestart)
│   └─ 정규장 사전 구독             ← 08:59 (timetable.krx_pre_subscribe)
├─ ─────────────────────────────
├─ 1일봉차트 자동다운로드           ← 20:40 (토글 + 시간)
├─ ─────────────────────────────
└─ 참고: 거래소 고정 시간 (변경 불가)
    ├─ 08:00 NXT 프리마켓 시작
    ├─ 09:00 정규장 시작
    ├─ 15:20 정규장 종료
    ├─ 15:30 종가 동시호가 종료
    ├─ 15:40 NXT 애프터마켓 시작
    ├─ 18:00 애프터마켓 지속 전환
    └─ 20:00 장마감
```

**배치 원리**:
- 상단: 사용자가 자주 조정하는 매매 시간대 (정규장 09:00~15:00)
- 중간: 장 시작 전 사전 준비 (07:58~08:59) — 고급 사용자용, 시간순 정렬
- 하단: 장마감 후 자동다운로드 (20:40)
- 최하단: 거래소 고정 시간 참고 (읽기전용) — 사용자 설정과 비교 기준

## 4. 아키텍처 원칙 준수

| 원칙 | 준수 내용 |
|---|---|
| **P10 (SSOT)** | 백엔드 설정 키는 변경 없이 UI 배치만 이동. 동일 키가 두 탭에서 중복 관리되지 않도록 이동 후 원래 탭에서는 완전 제거 |
| **P16 (살아있는 경로)** | 이동된 모든 설정 행은 새 탭에서 실제 렌더링·저장 경로에 연결. dead code(이동 후 원래 탭에 잔존하는 미사용 렌더 함수) 금지 |
| **P20 (폴백 금지)** | 이동 과정에서 설정 값 누락 시 빈 문자열/None으로 폴백하지 않음. 기존 `vals[key] ?? '기본값'` 패턴 유지 (이는 폴백이 아닌 명시적 기본값) |
| **P21 (사용자 투명성)** | 거래소 고정 시간 참고를 시간 설정 탭 최하단에 배치하여 사용자 설정 시간과 나란히 비교 가능. "왜 이 시간은 바꿀 수 없지?" 의문 해소 |
| **P23 (일관성)** | 동일 주제(시간/UI표시/API)를 단일 탭에 통합. 기존 공통 컴포넌트(`createTimePairInput`, `createTimeSlot`, `createToggleBtn`, `createDescText`, `sectionTitle`) 재사용 — 신규 컴포넌트 생성 금지. 용어 사전 준수 ("자동매수" 등 기존 라벨 유지) |
| **P24 (단순성)** | 탭당 단일 주제, 자동매매 탭 행 수 축소 (10+ → 5). 신규 "화면 설정" 탭은 플래시 효과 1개를 위해 탭 1개 추가하는 것은 과잉이므로 기각 — 자동매매 탭 하단으로 이동. 함수 50줄 이하, 파일 500줄 이하 기준 유지 검토 필요 (현재 general-settings.ts 약 1122줄 → 재구성 후에도 분할 필요성 별도 검토) |

## 5. UI 동작 설계

### 5-1. 자동매수/매도 토글과 시간 입력의 분리

재구성 전: 자동매수 토글과 시간쌍 입력이 한 행에 있었음 (토글 OFF 시 시간 입력 비활성화)

재구성 후:
- **자동매매 탭**: 자동매수 ON/OFF 토글만 (시간 입력 없음)
- **시간 설정 탭**: 자동매수 시간쌍 입력만 (토글 없음)

**연동 문제**: 자동매수 토글이 OFF일 때 시간 설정 탭의 시간쌍 입력을 비활성화할 것인가?

**설계 결정**: **시간 입력은 항상 활성화 유지** (비활성화하지 않음)
- 이유 1: 시간 설정 탭은 "시간 설정"이 주 목적이므로, 자동매매 탭의 토글 상태와 무관하게 시간 값 자체는 항상 편집 가능해야 함 (P24 단순성 — 탭 간 의존성 최소화)
- 이유 2: 사용자가 "자동매수는 끄되, 시간은 미리 설정해두고 싶을 때" 자연스러운 동작
- 이유 3: 백엔드는 `auto_buy_on`과 `buy_time_start`/`buy_time_end`를 독립 키로 관리하므로 UI도 독립성 유지 (P10 SSOT 부합)
- 단, 시간 설정 탭의 자동매수 시간쌍 행 하단에 "자동매수가 꺼져 있으면 이 시간은 사용되지 않습니다" 안내 문구 표시 (P21 투명성 — 사용자가 토글 상태를 시간 설정 탭에서도 인지)

### 5-2. 1일봉차트 자동다운로드 토글과 시간의 통합

재구성 전: API 설정 탭에서 토글과 시간 슬롯이 한 행에 있었음

재구성 후: 시간 설정 탭에서 동일하게 한 행에 유지 (패턴 일관성 — P23)
- 토글 OFF 시 시간 슬롯 비활성화 (기존 동작 유지)
- 안내 문구: "장마감 후 자동 다운로드 시간 (기본값 20:40) — OFF 시 수동 다운로드만 가능" (기존 문구 그대로 이동)

### 5-3. 실시간 현재가 플래시 효과 이동 (API 설정 → 자동매매 탭 하단)

자동매매 탭 하단에 새 섹션 "화면 표시" 추가:

```
[자동매매 탭 하단]
├─ (기존 매매 토글 행들)
└─ 화면 표시
    └─ 실시간 현재가 플래시 효과  [토글]
       실시간 시세 변경 시 노란색 플래시 깜빡임 효과 적용 여부
```

- 이유: "화면 설정" 탭 신설은 사용자가 과잉으로 판단하여 기각. 플래시 효과는 "실시간 시세 표시"와 관련이 있으므로 자동매매 화면의 부가 설정으로 자동매매 탭 하단에 배치하는 것이 가장 자연스러움
- 단일 토글이므로 자동매매 탭의 복잡도를 크게 증가시키지 않음 (P24)

### 5-4. 거래소 고정 시간 참고 표시

시간 설정 탭 최하단에 기존과 동일한 읽기전용 박스로 배치 (변경 없음, 위치만 이동).
- 사용자 설정 시간(상단)과 거래소 고정 시간(하단)을 한 탭에서 비교 가능 → P21 강화

## 6. 백엔드 영향

**백엔드 변경 없음**. 본 재구성은 순수 프론트엔드 UI 배치 변경:

### 7-2. 예상 신규 함수

### 7-3. 예상 검증 항목

- 각 탭의 설정 저장 동작 (기존과 동일하게 동작해야 함)
- 시간 설정 탭에서 자동매수/매도 시간쌍 변경 시 백엔드에 정상 저장
- 시간 설정 탭에서 1일봉 다운로드 시간·토글 변경 시 백엔드에 정상 저장
- 자동매매 탭에서 플래시 효과 토글 정상 동작

### 7-4. 예상 단계 분할 (참고용 — 2세션에서 확정)

1. **Step 1**: `TabId` 타입 확장 + 탭 순서 변경 + 빈 `renderTimeSettingsTab()` 골조 + 탭 전환 동작 검증
2. **Step 2**: 시간 설정 탭에 자동매수/매도 시간쌍 이동 + 자동매매 탭에서 해당 행 제거
3. **Step 3**: 시간 설정 탭에 장 시작 전 사전 준비 시간 3개 슬롯 + 거래소 고정 시간 참고 이동
4. **Step 4**: 시간 설정 탭에 1일봉차트 자동다운로드 이동 + API 설정 탭에서 제거
5. **Step 5**: 자동매매 탭 하단에 플래시 효과 이동 + API 설정 탭에서 제거
6. **Step 6**: 최종 검증 (빌드 + 브라우저 + 모든 설정 저장 동작)

> 각 Step은 규칙 0-1(세션당 1단계)에 따라 별도 세션에서 진행. 단, 작업량에 따라 2세션에서 통합/분할 가능.

## 8. 리스크 및 고려사항

### 8-1. 파일 길이 (P24)

현재 `general-settings.ts`는 약 1122줄. 재구성 후에도 크게 줄지 않을 수 있음 (이동이지 삭제가 아니므로).
- 2세션 심층 사전조사에서 파일 분할 필요성 별도 검토 (예: `general-settings/auto-trade-tab.ts`, `general-settings/time-settings-tab.ts` 등으로 분할)
- 단, 분할은 본 재구성과 별개 작업이므로 본 설계서 범위에서는 "검토 필요"로만 기록

### 8-2. 모듈 상태 변수 재배치

현재 모듈 상태 변수가 탭별로 그룹화되어 있으나, 시간 설정 탭 신설로 인해 변수 소속 재배치 필요:
- `buyTimeHandle`, `sellTimeHandle` → 자동매매 탭 참조에서 시간 설정 탭 참조로 이동
- `confirmedDlSlot`, `confirmedDlToggle`, `confirmedDlH/M`, `savingConfirmedDl` → API 설정 탭 참조에서 시간 설정 탭 참조로 이동
- `uiFlashToggle` → API 설정 탭 참조에서 자동매매 탭 참조로 이동
- `timetableResetSlot/WsSlot/KrxSlot` 및 관련 변수 → 자동매매 탭 참조에서 시간 설정 탭 참조로 이동

### 8-3. 자동매수/매도 토글과 시간 입력 분리의 사용자 혼란 가능성

자동매매 탭에서 "자동매수 ON"을 끈 후, 시간 설정 탭에서 시간을 변경하면 "이 시간이 실제로 적용되나?" 혼란 가능.
- 완화: 시간 설정 탭의 자동매수/매도 시간쌍 행 하단에 안내 문구 표시 (5-1절 설계 결정)
- 추가 완화 검토: 자동매수 토글 OFF 시 시간 설정 탭의 시간쌍 행 옆에 "현재 OFF" 칩 표시 (2세션에서 결정)

### 8-4. 텔레그램 탭 변경 없음

텔레그램 탭은 본 재구성에서 변경 없음. 순서만 4위에서 5위로 변경.
