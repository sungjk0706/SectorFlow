### 1-4. 기존 공통 자산 확인 (P23 사전 절차)

재구성에 필요한 모든 UI 요소는 기존 공통 자산으로 충분:
- 시간쌍 입력 → `createTimePairInput` (기존 사용)

### 2-2. 단계 분할 (6 Step × 1세션)

> 규칙 0-1(세션당 1단계) 준수. 각 Step은 독립적으로 완료·검증 가능한 크기로 분할.

#### Step 1: 탭 골조 추가 (A, B, C, D, E)
- `TabId`에 `'time-settings'` 추가
- `renderTabBar()`의 `tabs` 배열 순서 변경: 자동매매 → 시간 설정 → 투자모드 → API 설정 → 텔레그램
- 신규 `renderTimeSettingsTab(container)` 함수 골조 (섹션 제목 "시간 설정" + 빈 컨테이너만)
- `mount()`에 5번째 패널 추가 + 패널 순서 변경
- `unmount()`에 신규 변수 초기화 (이 시점에서는 시간 설정 탭 참조 변수가 없으므로 실제 추가는 Step 2~에서)
- **검증**: 빌드 + 브라우저에서 5개 탭 순서·전환 정상 (시간 설정 탭은 빈 화면)

#### Step 2: 자동매수/매도 시간쌍 이동 (F, G, N-부분, O-부분)
- `renderTimeSettingsTab()`에 자동매수 시간쌍 + 자동매도 시간쌍 행 추가 (기존 `renderAutoTradeTab()`의 211-294줄 코드 이동)
- 자동매매 탭에서 시간쌍 행 제거, 토글만 남김 (자동매수 토글·자동매도 토글)
- 토글 ON/OFF와 시간 입력 분리: 시간 입력은 항상 활성화 (설계서 5-1절)
- 시간 설정 탭의 자동매수/매도 시간쌍 행 하단에 안내 문구 추가 ("자동매수가 꺼져 있으면 이 시간은 사용되지 않습니다")
- 모듈 상태 변수 `buyTimeHandle`, `sellTimeHandle`을 시간 설정 탭 참조로 이동
- `syncFromSettings()`에서 자동매수/매도 시간쌍 동기화 부분을 시간 설정 탭 블록으로 이동
- **검증**: 빌드 + 브라우저에서 자동매매 탭(토글만)·시간 설정 탭(시간쌍) 정상 표시 + 시간쌍 변경 시 백엔드 저장 확인

#### Step 3: 장 시작 전 사전 준비 시간 + 거래소 고정 시간 이동 (H, I, N-부분, O-부분)
- `renderTimeSettingsTab()`에 "장 시작 전 사전 준비 시간" 섹션 + 타임테이블 3개 슬롯 추가 (기존 321-374줄 코드 이동)
- `renderTimeSettingsTab()`에 거래소 고정 시간 참고 박스 추가 (기존 376-412줄 코드 이동)
- 자동매매 탭에서 해당 섹션 전체 제거
- 모듈 상태 변수 `timetableResetSlot/WsSlot/KrxSlot` 및 관련 H/M 변수를 시간 설정 탭 참조로 이동
- `syncFromSettings()`에서 타임테이블 3개 키 동기화 부분을 시간 설정 탭 블록으로 이동
- **검증**: 빌드 + 브라우저에서 시간 설정 탭에 사전 준비 시간 3개 슬롯 + 거래소 고정 시간 참고 정상 표시 + 슬롯 변경 시 백엔드 저장 확인

#### Step 4: 1일봉차트 자동다운로드 이동 (J, K, N-부분, O-부분)
- `renderTimeSettingsTab()`에 1일봉차트 자동다운로드 행(토글 + 시간 슬롯) 추가 (기존 701-740줄 코드 이동)
- API 설정 탭에서 해당 행 제거
- 모듈 상태 변수 `confirmedDlSlot`, `confirmedDlToggle`, `confirmedDlH/M`, `savingConfirmedDl`를 시간 설정 탭 참조로 이동
- `syncFromSettings()`에서 1일봉 다운로드 동기화 부분을 시간 설정 탭 블록으로 이동
- **검증**: 빌드 + 브라우저에서 시간 설정 탭에 1일봉 다운로드 정상 표시 + 토글/시간 변경 시 백엔드 저장 확인 + API 설정 탭에서 제거 확인

#### Step 5: 플래시 효과 이동 (L, M, N-부분, O-부분)
- 자동매매 탭 하단에 "화면 표시" 섹션 + 플래시 효과 토글 행 추가 (기존 743-764줄 코드 이동)
- API 설정 탭에서 플래시 효과 행 제거
- 모듈 상태 변수 `uiFlashToggle`을 자동매매 탭 참조로 이동
- `syncFromSettings()`에서 플래시 효과 동기화 부분을 자동매매 탭 블록으로 이동
- **검증**: 빌드 + 브라우저에서 자동매매 탭 하단에 플래시 효과 정상 표시 + 토글 변경 시 백엔드 저장 확인 + API 설정 탭에서 제거 확인

#### Step 6: 최종 검증 + 정리 (P)
- 전체 빌드 (TypeScript 컴파일)
- 브라우저에서 5개 탭 전체 동작 확인:
  - 자동매매 탭: 마스터 토글, 자동매수 토글, 자동매도 토글, 체결 불가 시간대 차단, 플래시 효과
  - 시간 설정 탭: 자동매수/매도 시간쌍, 사전 준비 3슬롯, 1일봉 다운로드, 거래소 고정 시간 참고
  - 투자모드 탭: 기존과 동일
  - API 설정 탭: 증권사 선택 + API 키만
  - 텔레그램 탭: 기존과 동일
- 모든 설정 저장 동작 확인 (각 탭에서 변경 시 백엔드에 정상 저장)
- dead code 잔존 확인 (이동 후 원래 탭에 미사용 코드 없는지)
- 코드 리뷰: P10/P16/P20/P21/P23/P24 준수 최종 점검
- **검증**: 모든 항목 이상 없으면 Step 6 완료
