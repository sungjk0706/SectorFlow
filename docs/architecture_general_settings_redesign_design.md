# 설계서: 일반설정 탭 재분류 + 토글 이동 + 파일 분할

> **상태**: 설계 완료 (구현 대기)
> **작성일**: 2026-07-24
> **관련 원칙**: P10(SSOT) · P21(사용자 투명성) · P23(일관성) · P24(단순성) · P25(격리된 실패)
> **관련 파일**: `frontend/src/pages/general-settings.ts` (1443줄) · `frontend/src/components/common/*` (공통 자산)
> **관련 항목**: UI 개선 항목2(토글 이동) + 항목3(화면 표시 이동) + 항목4(탭 재분류) + F-04(파일 분할)

## 1. 배경 및 목표

### 1.1 현재 상태

일반설정 화면(`general-settings.ts`, 1443줄)은 5개 탭으로 구성:

| 탭 | 포함 내용 |
|---|---|
| 자동매매 | 마스터 스위치, 자동매수 토글, 자동매도 토글, 전역매매설정(안전장치), 화면 표시(플래시), 실시간 뉴스 설정(키워드·TTL) |
| 시간 설정 | 자동매수 시간, 자동매도 시간, 사전 준비 시간, 1일봉 자동다운로드, 거래소 고정 시간, 구독 한도 |
| 투자모드 | 테스트/실전 전환, 가상 투자금 |
| API 설정 | 증권사 선택, API 키 |
| 텔레그램 | 봇 토큰, 명령어 표 |

**문제점 3가지:**

1. **토글·시간 분리 (항목2)**: 자동매수/매도 토글은 "자동매매" 탭에, 시간은 "시간 설정" 탭에 분리. 사용자가 매수를 켜려면 자동매매 탭 → 토글 ON → 시간 설정 탭 이동 → 시간 확인/수정으로 2탭 왕복 필요.
2. **자동매매 탭 혼합 (항목3+4)**: 자동매매 탭에 매매 토글 + 안전장치 + 화면 표시(플래시) + 뉴스 설정이 섞여 탭 목적이 불명확. 화면 표시와 뉴스 설정은 매매와 직접 무관한 독립 설정.
3. **파일 과대 (F-04)**: 1443줄 단일 파일로 유지보수 어려움. 탭별 독립 파일 분할 필요.

### 1.2 목표

1. **토글·시간 통합 (항목2)**: 자동매수/매도 토글을 시간 설정 탭의 시간 라벨 우측으로 이동. 한 행에서 토글 ON/OFF + 시간 설정을 함께 조작.
2. **자동매매 탭 상태 배지 (P21 보완)**: 토글이 시간 설정 탭으로 옮겨가도 자동매매 탭에서 자동매수/매도 켜짐/꺼짐을 작은 배지로 확인 가능.
3. **탭 재분류 (항목3+4)**: 뉴스 설정 탭 + 화면 설정 탭 신설. 자동매매 탭은 마스터 스위치 + 안전장치만으로 매매 전용 탭 명확화.
4. **파일 분할 (F-04)**: 탭별 별도 파일로 분할하여 500줄 이하 유지 (P24).
5. **P10/P21/P23/P24/P25 완전 부합**.

### 1.3 비목표

- **설정 키/백엔드 변경 없음**: 본 설계는 프론트엔드 UI 구조 변경만. 백엔드 설정 키·API·WS 이벤트는 그대로.
- **매수설정 페이지(buy-settings.ts) 변경 없음**: 매수설정은 별도 페이지. 본 설계 범위 외.
- **새 기능 추가 없음**: 탭 재분류 + 이동 + 파일 분할만. 신규 설정 항목 없음.

## 2. 설계 방향

### 2.1 탭 재분류 설계안 비교

| 항목 | 옵션 A (선택) | 옵션 B | 옵션 C |
|---|---|---|---|
| 자동매매 탭 | 마스터 + 안전장치 + 상태 배지 | 마스터 + 안전장치 + 뉴스 설정 | 마스터 + 안전장치 + 화면 표시 + 뉴스 설정 (현행 유지) |
| 시간 설정 탭 | 시간 + 토글 함께 | 시간 + 토글 함께 | 시간 + 토글 함께 |
| 뉴스 설정 탭 | 신설 (키워드 + TTL) | 자동매매 탭에 유지 | 현행 유지 (자동매매 탭) |
| 화면 설정 탭 | 신설 (플래시 효과) | 신설 (플래시 효과) | 없음 |
| 탭 개수 | 5 → 7 | 5 → 6 | 5 (유지) |
| 장점 | 탭 목적 가장 명확, 확장성 | 탭 증가 최소, 뉴스=매매 연관 | 변경 최소 |
| 단점 | 탭 7개 | 자동매매 탭에 뉴스 잔류 | 자동매매 탭 혼합 지속 |

**선택: 옵션 A** — 탭 목적 명확성과 향후 확장성(화면 설정·뉴스 설정에 새 항목 추가 용이) 우선.

### 2.2 자동매매 탭 상태 표시 설계안 비교

| 항목 | 작은 배지 (선택) | 읽기 전용 토글 | 표시 제거 |
|---|---|---|---|
| 형태 | '켜짐'/'꺼짐' 색상 배지 (클릭 불가) | 토글 모양, 클릭 무반응 | 표시 없음 |
| P21 부합 | O (상태 확인 가능) | O | X (상태 알 수 없음) |
| 혼란 위험 | 낮음 | 클릭해도 안 바뀌어 혼란 | 낮음 (단, P21 위반) |

**선택: 작은 배지** — 클릭 불가 명확, P21 충족, 시각적 혼란 최소.

### 2.3 파일 분할 설계안 비교

| 항목 | 탭별 파일 분할 (선택) | 별도 세션 진행 | 분할 제외 |
|---|---|---|---|
| 시점 | 이 워크플로우에 통합 | 탭 재분류 후 별도 세션 | 분할 안 함 |
| 장점 | 중복 작업 회피 (탭 구조 변경 시 한 번에) | 구현 단순 | 작업량 최소 |
| 단점 | 구현 세션 수 증가 | 나중에 탭 구조 재취급 | 1443줄 유지, 유지보수 어려움 |

**선택: 탭별 파일 분할 통합 진행** — 탭 구조가 바뀌는 시점에 파일도 나누면 중복 작업 회피 (P24 단순성).

## 3. 선택안 동작 원리

### 3.1 최종 탭 구조 (7개 탭)

> **파일 분할 패턴**: 기존 `profit-*` 분할 패턴 준수 (P23 일관성) — `pages/` 폴더에 평행 파일, 하위 폴더 아님. 메인 파일 + `-{역할}` 접미사 파일 구조.

| 순서 | 탭 | 파일 | 포함 내용 |
|---|---|---|---|
| 1 | 자동매매 | `general-settings-auto-trade-tab.ts` | 마스터 스위치, 자동매수 상태 배지(읽기 전용), 자동매도 상태 배지(읽기 전용), 전역매매설정(안전장치) |
| 2 | 시간 설정 | `general-settings-time-settings-tab.ts` | 자동매수 시간+토글(한 행), 자동매도 시간+토글(한 행), 사전 준비 시간, 1일봉 자동다운로드, 거래소 고정 시간, 구독 한도 |
| 3 | 뉴스 설정 | `general-settings-news-settings-tab.ts` | 호재 키워드 칩, 뉴스 가산점 유지 시간 |
| 4 | 화면 설정 | `general-settings-display-settings-tab.ts` | 실시간 현재가 플래시 효과 |
| 5 | 투자모드 | `general-settings-account-tab.ts` | 테스트/실전 전환, 가상 투자금 |
| 6 | API 설정 | `general-settings-api-settings-tab.ts` | 증권사 선택, API 키 |
| 7 | 텔레그램 | `general-settings-telegram-tab.ts` | 봇 토큰, 명령어 표 |

**메인 파일**: `general-settings.ts` — 탭 바 렌더링, 탭 패널 전환, 설정 구독, 탭 모듈 호출 (200줄 이하 예상).
**공통 파일**: `general-settings-shared.ts` — 모듈 상태, GS 상수, 공통 헬퍼 (shouldForceOff, createHolidayBadge, updateHolidayBadges, scheduleTimetableSave).

### 3.2 자동매수/매도 시간+토글 통합 행 (시간 설정 탭)

```
자동매수 시간        [09:00 - 15:00]  [토글 ON/OFF]
자동매도 시간        [09:00 - 15:00]  [토글 ON/OFF]
```

- 시간 라벨 좌측, 시간쌍 입력 중앙, 토글 우측 (기존 `buildBuyTimeRow`/`buildSellTimeRow`에 토글 추가).
- 토글 OFF 시에도 시간 입력 활성화 유지 (현행 설계서 2-1 유지 — P24 탭 간 의존성 최소화).
- 토글 클릭 시 `auto_buy_on`/`auto_sell_on` 저장 (기존 `buildAutoBuyRow`/`buildAutoSellRow` 로직 이동).

### 3.3 자동매매 탭 상태 배지

```
자동매매              [마스터 토글]
  자동매수            [켜짐] (회색 배지, 클릭 불가)
  자동매도            [꺼짐] (회색 배지, 클릭 불가)
```

- `auto_buy_on`/`auto_sell_on` 값 기반 배지 표시 (P10 SSOT — uiStore에서만 참조).
- 배지 색상: 켜짐=기존 `COLOR.up`/`COLOR.upBg`, 꺼짐=중립 회색 (기존 표준 색상 재사용 — P23).
- `syncFromSettings`에서 배지 텍스트/색상 업데이트.

### 3.4 파일 분할 구조

> 기존 `profit-*` 분할 패턴 준수 (P23) — `pages/` 폴더에 평행 파일.

```
frontend/src/pages/
  general-settings.ts                    (메인: 탭 바, 전환, 구독, 모듈 호출)
  general-settings-shared.ts             (모듈 상태, GS 상수, 공통 헬퍼)
  general-settings-auto-trade-tab.ts     (자동매매 탭 렌더링 + 상태 배지)
  general-settings-time-settings-tab.ts  (시간 설정 탭 + 토글 통합)
  general-settings-news-settings-tab.ts  (뉴스 설정 탭)
  general-settings-display-settings-tab.ts (화면 설정 탭)
  general-settings-account-tab.ts        (투자모드 탭)
  general-settings-api-settings-tab.ts   (API 설정 탭)
  general-settings-telegram-tab.ts       (텔레그램 탭)
```

- **공통 상태 공유**: `general-settings-shared.ts`에 모듈 상태(`vals`, `settingsMgr`, `isTradingDay` 등)와 공통 헬퍼를 배치. 각 탭 파일은 `shared.ts`에서 import.
- **P10 SSOT**: 설정 값(`vals`)은 `shared.ts` 단일 소스. 각 탭은 참조만.
- **P25 격리된 실패**: 각 탭 렌더링 실패 시 해당 탭만 영향, 다른 탭 정상 (try/catch + 로깅).
- **상태 전달 패턴**: 기존 `profit-*` 분할이 상태 객체를 인자로 전달하는 패턴(F-05-a)을 준수. 단, general-settings는 모듈 상태가 파일 간 공유되므로 `shared.ts`에서 export하는 방식이 더 적합 (profit-*는 페이지 인스턴스별 상태, general-settings는 싱글톤 페이지 상태).

## 4. 표준 검토 근거

### P10 (SSOT)
- 설정 값 `vals`는 `shared.ts` 단일 소스. 각 탭 파일에서 독립 복사본 유지 금지.
- `auto_buy_on`/`auto_sell_on` 상태 배지는 uiStore에서만 참조 (중복 상태 관리 금지).

### P21 (사용자 투명성)
- 자동매수/매도 토글이 시간 설정 탭으로 이동해도 자동매매 탭에서 상태 배지로 켜짐/꺼짐 확인 가능.
- 사용자가 "왜 매수가 안 되지?" 할 때 자동매매 탭에서 토글 꺼짐을 바로 인지.

### P23 (일관성)
- 기존 공통 자산 재사용: `createToggleBtn`, `createTimePairInput`, `sectionTitle`, `createDescText`, `createTabBar`, `COLOR.*` 표준 색상.
- 파일 분할 패턴: 기존 `buy-settings.ts`(430줄 단일 파일)와 달리 탭별 분할이지만, 메인 파일 + 하위 모듈 구조는 프로젝트 일반 패턴 준수.
- 용어 사전 준수: "자동매매", "자동매수", "자동매도", "시간 설정", "뉴스 설정", "화면 설정" (부록 L).

### P24 (단순성)
- 파일 분할로 각 파일 500줄 이하 유지 (현재 1443줄 → 메인 200줄 + 각 탭 100~300줄 예상).
- 토글·시간 통합 행으로 2탭 왕복 제거 (조작 단순화).
- 탭 재분류로 각 탭 목적 명확 (혼합 제거).

### P25 (격리된 실패)
- 각 탭 파일 렌더링 실패 시 해당 탭만 영향, 다른 탭 정상 작동.
- `buildTabPanels`에서 각 탭 렌더링을 try/catch로 격리, 실패 시 로깅 + 폴백 메시지 표시 (silent pass 금지).
