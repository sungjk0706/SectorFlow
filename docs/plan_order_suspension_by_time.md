> **상태**: 사전조사 완료 · 구현 계획 수립 완료 · **사용자 승인 대기**

> **관련 원칙**: P10(SSOT) · P15(단일 주문 경로) · P16(살아있는 경로) · P20(폴백 금지) · P21(사용자 투명성) · P23(일관성) · P24(단순성)

## 1. 배경 및 목적

### 1-1. 문제 상황
- 본 앱은 주문 방식을 **시장가 단일 운용** (`trading.py` execute_buy/execute_sell 내 `order_type = "시장가"`)
- 실시간 체결이 불가능한 시간대에 시장가 주문이 들어가면 **미체결 또는 오류** 발생 가능
- 현재 일부 시간대는 차단되어 있으나 **누락된 구간이 존재** (사전조사에서 발견)

### 1-2. 목적
- 체결 불가 시간대에 매수/매도 주문을 자동으로 거부
- 체결 가능 시간대가 되면 별도 재개 로직 없이 자동으로 주문 통과 (시간 기반이므로)
- 사용자에게 "왜 주문이 안 들어가지?" 의문을 갖지 않도록 UI에 상태 표시 (P21)

## 2. 사전조사 결과 (심화)

### 2-1. 기존 시간 체계 SSOT (재사용 대상 — P23 공통 자산)

**파일**: `backend/app/services/daily_time_scheduler.py`

이미 KRX/NXT 시간대 상수와 `calc_timebased_market_phase()` SSOT가 구현되어 있음.
**새 시간 상수를 만들지 말고 기존 `market_phase` 값을 활용** (P10 SSOT).

#### KRX market_phase 값과 시간대 매핑
| 시간대 | market_phase["krx"] | 실시간 체결 | 비고 |
|---|---|---|---|
| 00:00~08:00 | 장개시전 | 불가 | |
| 08:00~08:30 | 장전 대기 | 불가 | |
| 08:30~08:40 | 장전 시간외 | 가능 (선착순) | 시장가 아닌 전일 종가 거래 |
| 08:40~08:50 | 동시호가 접수 | 불가 | 주문 접수만, 체결 없음 |
| **08:50~09:00** | **시가 동시호가** | **불가 (09:00 일괄)** | **현재 차단 누락** |
| 09:00~15:20 | 정규장 | 가능 | |
| **15:20~15:30** | **종가 동시호가** | **불가 (15:30 일괄)** | **현재 차단 누락** |
| 15:30~15:40 | 체결 정산 | 불가 | 현재 `is_krx_after_hours()` 차단 |
| 15:40~16:00 | 장후 시간외 | 가능 (선착순) | 시장가 아닌 당일 종가 거래 |
| 16:00~18:00 | 시간외 단일가 | 불가 (10분 단위) | 현재 차단 |
| 18:00~20:00 | 장 종료 | 불가 | 현재 차단 |
| 20:00~24:00 | 장마감 | 불가 | |

#### NXT market_phase 값과 시간대 매핑
| 시간대 | market_phase["nxt"] | 실시간 체결 | 비고 |
|---|---|---|---|
| 08:00~08:50 | 프리마켓 | 가능 | NXT 활성 |
| 08:50~09:00 | 정규장 준비 | 불가 | NXT 비활성 |
| 09:00~15:20 | 메인마켓 | 가능 | NXT 활성 |
| 15:20~15:30 | 조기 마감 | 불가 | NXT 비활성 |
| 15:30~15:40 | 단일가 매매 | 불가 (15:40 일괄) | NXT 비활성 |
| 15:40~18:00 | 애프터마켓 | 가능 | NXT 활성 |
| 18:00~20:00 | 애프터마켓 지속 | 가능 | NXT 활성 |

### 2-2. 현재 차단 로직 현황

#### 기존 함수: `is_krx_after_hours()` (daily_time_scheduler.py:213-227)
```python
return krx in ("체결 정산", "장후 시간외", "시간외 단일가", "장 종료")
```
- **차단 O**: 체결 정산, 장후 시간외, 시간외 단일가, 장 종료
- **차단 X (누락)**: 시가 동시호가(08:50~09:00), 종가 동시호가(15:20~15:30), 동시호가 접수(08:40~08:50)

#### 기존 함수: `KRX_INACTIVE_PHASES` (daily_time_scheduler.py:169-173)
```python
frozenset({"장개시전", "장전 대기", "장전 시간외", "동시호가 접수", "시가 동시호가",
           "종가 동시호가", "체결 정산", "장후 시간외", "시간외 단일가", "장 종료",
           "장마감", "휴장일"})
```
- `is_nxt_only_window()`에서 사용 — KRX 비활성 + NXT 활성 판별용
- **"시가 동시호가", "종가 동시호가"가 이미 포함되어 있음** — 이 집합을 재사용하면 누락 없이 차단 가능

### 2-3. 주문 경로별 시간 체크 현황 (중요 — P15/P16)

| 주문 함수 | 시간 체크 위치 | 문제점 |
|---|---|---|
| `execute_buy()` (trading.py:92) | **내부에 없음** | `buy_order_executor.py` 외부에서만 `is_krx_after_hours()` 호출. 수동 매수(force_buy) 시 우회 가능. **P16 위반 소지** |
| `execute_sell()` (trading.py:431) | **전혀 없음** | 매도는 시간 체크 자체가 없음. 동시호가 시간대에 매도 주문 들어갈 수 있음 |

#### 매수 호출 경로
1. `buy_order_executor.evaluate_buy_candidates()` → 외부에서 `is_krx_after_hours()` 체크 → `execute_buy()` 호출
2. 매수대기 수동 매수 → `execute_buy(force_buy=True)` 직접 호출 → **시간 체크 우회**

#### 매도 호출 경로
1. `trading.py` 내 손절/익절/T/S 조건 hit → `execute_sell()` 직접 호출 → **시간 체크 없음**

### 2-4. 설정 저장 구조
- DB 테이블: `integrated_system_settings` (key-value)
- 메모리 캐시: `engine_state.integrated_system_settings_cache` (P13 설정 메모리 상주)
- 기본값 정의: `backend/app/core/settings_defaults.py` → `DEFAULT_USER_SETTINGS`

### 2-5. UI 패턴 (P23 일관성)
- **설정 토글 패턴**: `general-settings.ts` 자동매매 탭 — `createToggleBtn()` 사용
- **헤더 칩 패턴**: `header.ts` — 서킷브레이커 칩 (빨간색, 클릭 시 해제)
- **WS 이벤트 패턴**: `binding.ts` → `uiStore.ts` → `header.ts` 구독 체인

## 3. 사용자 결정 필요 항목 (구현 전 반드시 확정)

### 3-1. NXT 종목 처리 방침 (★가장 중요한 결정)
제안하신 4개 시간대 중 15:30~15:40, 15:40~16:00은 **KRX는 비활성이지만 NXT는 활성**입니다.

| 시간대 | KRX | NXT | 질문 |
|---|---|---|---|
| 08:50~09:00 | 시가 동시호가 (불가) | 정규장 준비 (불가) | 양쪽 다 불가 → 전부 차단? |
| 15:20~15:30 | 종가 동시호가 (불가) | 조기 마감 (불가) | 양쪽 다 불가 → 전부 차단? |
| 15:30~15:40 | 체결 정산 (불가) | 단일가 매매 (불가) | 양쪽 다 불가 → 전부 차단? |
| 15:40~16:00 | 장후 시간외 (시장가 불가) | 애프터마켓 (가능) | **NXT 종목은 매매 허용? 차단?** |

**선택지**:
- **A) 전부 차단**: 시간대와 무관하게 4개 구간 전부 주문 중단. 가장 안전하지만 NXT 애프터마켓(15:40~16:00) 매매 기회 상실
- **B) KRX/NXT 분리 차단**: KRX 단독 종목은 4개 구간 전부 차단, NXT 활성 구간(15:40~16:00 애프터마켓)의 NXT 종목은 매매 허용. 기존 `is_nxt_only_window()` 패턴과 일관 (P23)

### 3-2. 시장가 vs 시간외 거래 구분
- 08:30~08:40 (장전 시간외), 15:40~16:00 (장후 시간외)는 **시장가가 아닌 전일/당일 종가 거래**만 가능
- 본 앱은 시장가 단일 운용이므로 이 구간도 시장가 주문은 실패
- **이 구간도 차단 대상에 포함할지?** (현재 `is_krx_after_hours()`는 장후 시간외를 차단하지만 장전 시간외는 차단 안 함)

### 3-3. ±5초 여유 시간 적용 방식
- **A) phase 산정 시점에 여유 적용**: `calc_timebased_market_phase()`에서 ±5초 버퍼. WS 구독 등 다른 기능에도 영향
- **B) 주문 체크 시점에만 여유 적용**: 주문 함수 내부에서만 ±5초 버퍼. 다른 기능에 영향 없음 (추천)

### 3-4. 수동 매수(force_buy) 시간대 차단 여부
- 현재 수동 매수는 자동매매 게이트만 우회하고 나머지 가드는 적용
- **동시호가 시간대에도 수동 매수를 차단할지?** 아니면 수동 매수는 사용자 책임으로 허용?

### 3-5. 매도 주문도 동시호가 시간대에 차단할지?
- 매도는 "리스크 축소 행위"라 기존 `check_sell_order_allowed()`는 서킷브레이커만 체크
- **동시호가 시간대 매도도 차단할지?** (시장가 매도도 동시호가에서는 체결 불가이므로 차단 권장)

## 7. 위험 및 주의점

1. **기존 `is_krx_after_hours()` 수정 시 영향 범위**
   - `buy_order_executor.py`에서 호출 중 — 수정 시 기존 매수 차단 동작 변화 가능
   - **대안**: 기존 함수는 유지하고 신규 함수 추가 (예: `is_order_blocked_by_time()`) — 기존 동작 보존

2. **NXT 분리 차단 시 복잡도 증가 (P24)**
   - 3-1 B 선택 시 KRX/NXT 분기 로직 추가 → 복잡도 상승
   - 기존 `is_nxt_only_window()` 패턴과 일관되게 유지 필요

3. **±5초 여유 적용 시 경계 오차**
   - 09:00:00 정각에 "시가 동시호가"에서 "정규장"으로 전환 — 08:59:55에 주문 시 차단? 허용?
   - phase 산정은 분 단위(`t = hour * 60 + minute`)이므로 초 단위 여유는 별도 계산 필요

4. **매도 차단 시 손절 기회 상실**
   - 동시호가 10분간 매도 차단 시 그 사이 손절 조건 hit되어도 매도 불가
   - 다음 체결 가능 시간(정규장 재개)까지 대기 — 사용자에게 사전 안내 필요

## 8. 사용자 승인 대기 항목

이 계획서는 **사용자 결정(섹션 3) + 승인** 후 구현 진행.

### 결정 필요 항목 (섹션 3 재조회)
1. **3-1 NXT 종목 처리**: A) 전부 차단 / B) KRX·NXT 분리 차단
2. **3-2 시간외 거래 구간**: 장전/장후 시간외 시장가 주문 차단 포함 여부
3. **3-3 ±5초 여유 적용**: A) phase 산정 / B) 주문 체크 시점 (추천 B)
4. **3-4 수동 매수 차단**: 동시호가 시간대 수동 매수도 차단 여부
5. **3-5 매도 차단**: 동시호가 시간대 매도도 차단 여부 (권장: 차단)

### 승인 트리거 대기
사용자가 "진행해", "구현해", "적용해" 등 명시적 실행 지시어를 줄 때까지 코드 수정 금지 (AGENTS.md 섹션3 규칙 0).
