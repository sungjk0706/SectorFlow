## 🛑 [필독] 코딩 AI 핵심 제약 사항 (Strict Rules)

1. **기술 스택 강제 (No Frameworks):** 
   - 이 프로젝트는 **순수 Vanilla TypeScript + Vite** 기반입니다.
   - 절대 React, Vue, jQuery 등의 외부 라이브러리를 임의로 `npm install` 하거나 임포트하지 마십시오.
2. **독단적 진행 금지 (Wait for Approval):** 
   - 각 Phase(단계)별 작업이 끝나면 **반드시 작업을 멈추고 사용자에게 결과를 보고**하십시오.
   - 사용자의 "다음 단계 진행해"라는 승인 없이는 절대 다음 Phase 코드를 수정하지 마십시오.
3. **코드 보호 (Preserve Existing Logic):**
   - 백엔드와 연동되는 웹소켓 로직(`engine_ws_dispatch.py`, `binding.ts` 등)의 핵심 데이터 구조를 마음대로 삭제하지 마십시오.
4. **테스트 의무 (Test Driven):**
   - 수정을 마친 후에는 반드시 `npm run test` 명령을 실행하여 기존의 47개 테스트 코드가 깨지지 않았는지 스스로 검증하고 결과를 보고하십시오.

## 🎯 작업 개요: 3대 아키텍처 개선 (Phases)

당신이 순차적으로 수행해야 할 프론트엔드 최적화 3단계입니다.

### [Phase 1] 다중 증권사(LS/키움) 전환 어댑터 구조 도입
- **목표:** 현재 하드코딩된 '키움증권(kiwoom)' 텍스트와 설정을 범용적인 증권사 선택 탭 구조로 개편.
- **수정 타겟:**
  - `src/layout/header.ts` & `header.ui.ts`: 상단 뱃지에서 'kiwoom' 종속 변수 분리.
  - `src/components/common/broker-badge.ts`: LS증권 테마(예: 빨간색 계열) 및 텍스트 추가.
  - `src/pages/general-settings.ui.ts`: 기존 '키움증권 앱키' 등을 `[키움증권] / [LS증권]` 탭 UI로 분리.
- **보고 사항:** UI가 어떻게 변경되었는지 시각적인 구조(HTML 구조)를 사용자에게 설명할 것.

### [Phase 2] 시스템 상태 표지판 (상태 머신 UI) 신설
- **목표:** 백엔드의 상태(`WAITING_FIRST_TICK`, `LIVE`)를 프론트엔드 상단에 신호등처럼 시각화.
- **수정 타겟:**
  - `src/stores/hotStore.ts`: `realtime-state` 이벤트 수신 시 상태값 저장 로직 추가.
  - `src/layout/header.ui.ts`: 우측 상단에 상태 표시줄(🟡 실시간 대기 중 / 🟢 가동 중) UI 렌더링.
- **보고 사항:** 이벤트 수신 시 UI가 어떻게 업데이트되는지 상태 흐름을 사용자에게 설명할 것.

### [Phase 3] UI 컴포넌트 표준화 (레고 블록화)
- **목표:** 흩어져 있는 DOM 조작 로직을 재사용 가능한 Web Component 형식이나 모듈화된 함수로 묶음 처리.
- **수정 타겟:**
  - `src/components/common/` 내의 반복적인 HTML 생성 로직 정비.
- **보고 사항:** 어떤 컴포넌트들이 재사용 가능해졌는지 리스트업하여 보고할 것.
